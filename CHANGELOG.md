# Changelog

All notable changes to this project will be documented in this file.

## [0.9.0] - 2026-04-22

### Added
- ChatGPT/Gemini API 키를 암호화 저장하고 연결 테스트와 모델 선택을 할 수 있는 `설정 > AI 관리` 탭을 추가했습니다.
- 화면 하단 고정 TeamScope 프롬프트를 추가해 대시보드, 개발자, 설정, 가이드 어디서든 AI 분석을 요청할 수 있게 했습니다.
- TeamScope AI 에이전트 오케스트레이션을 추가해 프롬프트 의도, 현재 대시보드 필터, 동기화 스냅샷, RAG 기준 문서를 조합해 답변하도록 했습니다.
- RAG 데이터를 `persona`, `project-management`, `teamscope` 디렉터리의 markdown 문서로 분리해 DORA, SPACE, PMI/EVM, 가동률, 스코어링 기준을 관리하기 쉽게 구성했습니다.
- 여러 달 범위의 공수/가동률 질문에 대해 월별 집계 데이터를 만들고, 답변 본문 안에 공수 막대와 가동률 선 그래프를 함께 보여주는 AI 차트를 추가했습니다.
- AI 답변 히스토리, 이어 질문 컨텍스트, RAG 디버그, 사용 모델과 토큰 사용량 표시를 추가했습니다.
- 로그인 데모 GIF와 poster에 AI 월별 분석 장면을 포함했습니다.

### Changed
- AI 답변 UI를 원문 덤프 방식이 아니라 섹션형 대화 카드, 수치 강조, 월별 차트 중심으로 재구성했습니다.
- 프롬프트 placeholder를 활성 Provider에 맞춰 랜덤으로 보여주고, 이모지와 문구 배치를 현재 프롬프트 UI에 맞게 다듬었습니다.
- DORA/PMI/SPACE는 답변에서 장황하게 설명하지 않고 내부 판단 기준으로 사용하며, 실제 답변은 바로 실행 가능한 운영 포인트만 짚도록 조정했습니다.
- 여러 달 범위가 들어오면 사용자가 `월별`이라고 쓰지 않아도 월별 비교를 기본 분석 구조로 사용하도록 바꿨습니다.
- AI 월별 공수/가동률은 Jira/GitLab API를 재호출하지 않고 TeamScope 대시보드가 사용하는 DB 동기화 스냅샷만 읽어 계산하도록 정리했습니다.

### Fixed
- AI 답변에서 빈 섹션, 과한 bullet, `**` 같은 markdown 장식, 중복 섹션명이 그대로 노출되던 문제를 줄였습니다.
- 월별 분석에서 전체 기간을 한 덩어리로 설명하거나 최고/최저 가동률을 잘못 비교할 수 있던 지침을 보강했습니다.
- 프롬프트를 통한 API 키 저장, 서버 데이터 수정/삭제, Jira/GitLab 원천 데이터 변경을 AI 에이전트 경로에서 수행하지 못하도록 보안 경계를 강화했습니다.
- Gemini 연결 후 프롬프트바의 모델 표시, placeholder 위치, 글래스 UI 윤곽이 어색하던 부분을 다듬었습니다.

## [0.8.0] - 2026-04-22

### Added
- 대시보드, 개발자, 설정, 가이드 이동 시 화면 중앙 로딩 오버레이를 추가하고, 랜덤 아이콘과 부드러운 안내 문구로 현재 이동 중임을 명확히 보여주도록 했습니다.
- 대시보드 프로젝트 필터를 개발자 필터와 같은 선택형 UI로 개편해 등록된 프로젝트를 바로 선택하고, 마지막으로 반영한 일자/개발자/프로젝트 상태가 유지되도록 했습니다.
- 프로젝트 기준 멤버 조회 결과에 페이지네이션을 추가해 조회 멤버가 많아도 끊어서 확인할 수 있게 했습니다.
- 프로젝트 제거 시 TeamScope 내부 스냅샷, 매핑, 점수 캐시를 함께 정리하는 내부 데이터 클린업 경로를 추가했습니다.

### Changed
- 설정 탭 순서를 `프로젝트 관리`, `그룹 관리`, `스코어링 가중치`, `계정 관리`로 재구성하고, 계정 관리는 다른 운영 설정과 구분되도록 마지막 영역으로 분리했습니다.
- 기존 멤버 매핑 탭에 있던 `프로젝트 기준 멤버 불러오기`와 `멤버 매핑`을 `프로젝트 관리` 탭 안으로 옮겨 프로젝트 연결부터 멤버 저장, 매핑, 동기화까지 한 흐름에서 처리하도록 바꿨습니다.
- 가이드 탭을 새 설정 구조에 맞게 다시 정리하고, 내부 미리보기 UI가 좁은 화면에서도 깨지지 않도록 단계형 로드맵과 간결한 프리뷰 카드로 개선했습니다.
- Owner 전용 `Test Harness` 항목을 주요 탭과 구분선으로 분리하고 `Test Harness (BETA)`로 표기했습니다.
- 대시보드 초기 로딩 시 중복 점수 조회를 줄이고, 이전 기간 비교는 summary-only 경로로 계산하며, Gantt 데이터는 주요 KPI 이후에 로드되도록 조정했습니다.
- Gantt 차트의 종료/닫힘 티켓 색상을 중간 슬레이트 톤으로 통일해 어두운 배경에서도 과하게 묻히거나 튀지 않게 했습니다.

### Fixed
- 대시보드에서 개발자와 프로젝트를 전체 선택한 뒤 반영했을 때 첫 번째 개발자만 적용되는 경우가 있던 필터 동기화 문제를 수정했습니다.
- 비활성 프로젝트의 Jira/GitLab 스냅샷이 대시보드, 점수, 개발자 상세, Gantt, Excel 내보내기에 섞일 수 있던 문제를 막고, 삭제된 프로젝트의 잔여 내부 데이터를 정리했습니다.
- 프로젝트 제거가 오래 걸리거나 저장 상태를 알기 어려웠던 흐름을 개선하고, 제거 시 TeamScope 내부 데이터만 정리하며 Jira/GitLab 원천 데이터는 절대 변경하지 않도록 경계를 명확히 했습니다.
- 프로젝트 선택 selectbox와 가이드 내부 카드가 기존 다크 UI와 어울리지 않거나 깨져 보이던 부분을 정리했습니다.

## [0.7.0] - 2026-04-03

### Added
- Owner 전용 `Flow Lab`(Test Harness) 화면을 추가해 프로젝트 연결, 멤버 해석, 스냅샷 동기화, 점수 계산, 대시보드 조회 경로를 단계별로 실행하고 확인할 수 있게 했습니다.
- 회귀 데이터셋 기반 검증, 실행 이력 조회, Inspector 기반 상세 분석을 제공해 운영 변경 이후에도 핵심 흐름을 반복 검증할 수 있게 했습니다.
- PostgreSQL 관측, 날짜 정규화, 인덱스 적용, 월간 요약 뷰 초기화/갱신을 위한 DB 운영 스크립트를 추가했습니다.

### Changed
- `dashboard-insights`와 `scores`가 문자열 날짜 대신 정규화된 날짜 컬럼을 우선 사용하도록 바꾸고, 무필터 월간 추세는 materialized view를 우선 읽도록 최적화했습니다.
- 사이드바를 접기/펼치기 가능하게 바꾸고, Owner에게는 `Test Harness` 바로가기를 함께 노출했습니다.
- README를 최신 운영 절차 기준으로 보완해 `Test Harness`, DB 최적화 명령, 분석 파이프라인 설명을 현재 코드와 맞췄습니다.

### Fixed
- Flow Lab 관련 API가 예외 상황에서도 일관된 JSON 오류 응답을 반환하도록 정리했습니다.
- Jira/GitLab 스냅샷과 점수 조회의 기간 필터가 정규화된 날짜 컬럼을 사용하도록 바꿔 월간 범위 해석 정확도를 높였습니다.
- 무필터 대시보드 월간 추세 조회 시 반복 집계 부담을 줄여 응답 편차를 완화했습니다.

## [0.6.0] - 2026-04-03

### Added
- Passkey 등록/삭제 관리와 비밀번호 변경 동선을 사이드바 하단으로 옮겨 계정 보안 기능을 헤더와 분리했습니다.
- 워크스페이스 초대 시 1회용 로그인 링크를 생성하고, 링크를 열면 계정 생성과 워크스페이스 자동 참여가 이어지는 초대 기반 온보딩을 추가했습니다.
- 초대 링크로 생성된 계정에 기본 credential 비밀번호를 함께 부여해 비밀번호 로그인과 Passkey 등록을 이어갈 수 있게 했습니다.
- Owner가 멤버와 대기 중인 초대를 권한 그룹별로 보고 역할을 직접 수정할 수 있는 접근 관리 화면을 보강했습니다.

### Changed
- 비밀번호 해시를 Argon2id 기반으로 강화하고, 기존 scrypt 계정도 계속 로그인할 수 있도록 점진적 검증 경로를 추가했습니다.
- Passkey 로그인 취소/미지원 상황에서 개발 오버레이 대신 사용자 안내 문구로 처리되도록 로그인 UX를 정리했습니다.
- 공개 로그인 화면의 Magic Link 발송 UI는 비활성화하고, 추후 지원 예정 흐름으로 정리했습니다.
- 로컬 실행 스크립트 이름을 `dev:mac`, `dev:windows`로 단순화했습니다.
- README를 사용자용 가이드와 개발자용 백서 구조로 전면 개편했습니다.

### Fixed
- Passkey 로그인 취소 시 `NotAllowedError`가 개발 오류처럼 보이던 문제를 완화했습니다.
- 워크스페이스 초대 이후 실제 로그인 가능한 계정이 자연스럽게 이어지지 않던 온보딩 공백을 보완했습니다.
- 버전 문서와 실행 스크립트 예시가 현재 프로젝트 상태와 어긋나던 부분을 정리했습니다.

## [0.5.0] - 2026-04-02

### Added
- 공개 페이지(`login`, `changelog`)와 로그인 보호 영역(`dashboard`, `developer`, `settings`, `guide`)을 분리한 라우트 구조를 도입해 진입 흐름을 더 명확히 만들었습니다.
- 기본 워크스페이스 자동 생성, 초대 자동 수락, 활성 워크스페이스 고정 로직을 추가해 Better Auth 기반 워크스페이스 운영을 안정화했습니다.
- 워크스페이스 멤버/사용자 조회 API와 설정 내 접근 관리 폼을 보강해 초대, 역할 변경, 계정 생성 흐름을 한 곳에서 관리할 수 있게 했습니다.
- macOS/Linux용 `dev-up.sh`와 Windows용 `dev-up.ps1`를 추가해 로컬 Postgres 준비부터 시드, 개발 서버 실행까지 한 번에 시작할 수 있게 했습니다.

### Changed
- README, changelog, 앱 내 릴리즈 노트가 현재 기능 세트와 같은 버전 기준을 바라보도록 정리했습니다.
- 앱 셸을 `(app)` 레이아웃 기준으로 재구성하고, 사이드바 버전 표기, 상단 로딩바, changelog 자동 오픈 흐름을 일관되게 맞췄습니다.
- 개발자 랭킹, 개발자 상세, 대시보드 인사이트가 같은 분석 경로와 필터 기준을 공유하도록 정리해 화면마다 수치가 달라 보이던 혼선을 줄였습니다.
- 설정 화면을 멤버 매핑, 프로젝트 관리, 그룹 관리, 스코어링 탭 중심으로 다듬고 GitLab 그룹 URL/토큰 기반 운영 설명을 강화했습니다.

### Fixed
- GitLab 그룹/프로젝트 식별 시 URL, 프로젝트 키, 그룹 경로 해석이 엇갈리던 문제를 보완해 연결 테스트와 동기화 성공률을 높였습니다.
- 프로젝트 멤버 조회와 동기화에서 하위 프로젝트 멤버가 빠지거나 기존 매핑이 반응하지 않던 문제를 완화했습니다.
- 개발자 상세, 순위, 공수 관련 화면에서 점수/설명 기준이 어긋나던 문제를 정리했습니다.

## [0.4.0] - 2026-04-02

### Added
- 가이드 페이지를 상단 4단계 고정 + 하단 스와이프형 안내 카드 구조로 개편해 처음 쓰는 사용자도 순서대로 따라가기 쉽게 만들었습니다.
- GitLab 그룹 URL + 그룹 액세스 토큰을 지원하고, 하위 프로젝트 멤버 및 MR까지 수집할 수 있는 그룹 단위 연동 흐름을 추가했습니다.
- 개발자 상세 화면도 대시보드/개발자 목록과 같은 분석 소스를 사용하도록 정리해, 동일 개발자 점수를 같은 기준으로 확인할 수 있게 했습니다.
- 프로젝트 관리 카드에 저장된 연결 정보를 즉시 다시 확인할 수 있는 연결 테스트 흐름을 보강했습니다.

### Changed
- 대시보드 개발자 순위, 개발자 목록, 개발자 상세의 점수 계산 경로를 `dashboard-insights` 기준으로 통일했습니다.
- GitLab 그룹 연동 시 전체 하위 프로젝트 데이터를 무조건 적재하지 않고, 프로젝트에 매핑된 개발자 기준으로 필요한 MR만 수집하도록 최적화했습니다.
- 공수활용률 안내 문구를 실제 계산 로직에 맞게 정리하고, Gantt의 현재 보이는 기간 공수 현황과 개발자 순위의 전체 기간 공수활용률이 헷갈리지 않게 설명을 보강했습니다.
- 가이드 페이지 단계 번호 배지를 더 넓은 정사각형 톤으로 통일하고, 현재 단계 카드가 형광 윤곽선으로 강조되도록 개선했습니다.
- 개발자 상세의 뒤로가기 동선을 대시보드가 아닌 개발자 목록으로 바로 돌아가도록 수정했습니다.

### Fixed
- 개발자 상세에서 GitLab 점수가 0점으로 보이던 문제를 보완해 목록/대시보드와 같은 값이 나오도록 수정했습니다.
- 개발자 순위와 개발자 탭이 서로 다른 API를 보던 문제를 해결해, 두 화면의 지표가 일치하도록 맞췄습니다.
- GitLab 그룹 멤버 조회 시 하위 프로젝트 멤버가 누락되어 기존 매핑만 선택해도 반응이 없던 문제를 보완했습니다.
